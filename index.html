SaltrUnitySdk Documentation

INTRODUCTION

SALTR Unity SDK is a library of classes which help to develop mobile games that are to be integrated with SALTR platform.
This SDK performs all necessary and possible action with SALTR API to connect, update, set and download data related to game's features or levels. All data received from SALTR API is parsed and represented through set of instances of classes, each carrying specific objects and their properties. Basically SDK, as the API, has few simple actions. The most important one is connecting (getAppData), which loads the app data objects containing features, experiments and level headers. This and other actions will be described in the sections below.

USAGE

To use the SDK you need to download SDK repository and then import files to your project.
The entry point is the SLTUnity class, which you can create and initialize in code. Or you can darg and drop the SALTR prefab on your main scene and access its instance through SaltrUnity component.
Note: All classes in the package start with "SLT" prefix.

STRUCTURE

The SDK has the following directory structure:
Assets/Libs/Saltr - folder contains the SALTR prefab
Saltr.UnitySdk - main namespace of library, contains main classes
Saltr.UnitySdk.ApiCall - classes related to SALTR api calls wihch are made after SaltrUnitySdk is running
Saltr.UnitySdk.Domain - contains Domain Model classes
Saltr.UnitySdk.Domain.Model.Canvas - classes related to canvas based games;
Saltr.UnitySdk.Domain.Model.Matrix - classes related to matrix based games;
Saltr.UnitySdk.Network - network related classes (DownloadManager, etc..)
Saltr.UnitySdk.Repository - local data repository classes (implementation widely varies through platforms);
Saltr.UnitySdk.Utils - contains Domain Model classes
